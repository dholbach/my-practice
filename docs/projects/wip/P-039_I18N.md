# P-039: Django i18n — Bilingual App UI

**Status**: WIP
**Priority**: Medium (contributor on-ramp, driven by external interest)
**Depends on**: P-038 ✅, P-024 ✅

## Goal

Make all user-facing UI text translatable via Django's i18n framework so the app
can run in German or English (or any future language) without code changes.

## Scaffolding (done in feat/i18n-scaffold)

- [x] `LocaleMiddleware` added to `MIDDLEWARE` (after `SessionMiddleware`)
- [x] `LANGUAGES`, `LOCALE_PATHS` added to `settings.py`
- [x] `django.template.context_processors.i18n` added
- [x] `path("i18n/", include("django.conf.urls.i18n"))` — enables `set_language` view
- [x] `gettext` added to Dockerfile (required for `makemessages`/`compilemessages`)
- [x] `app/locale/de/LC_MESSAGES/` and `app/locale/en/LC_MESSAGES/` directories created

- [x] `{% load i18n %}` + dynamic `lang=` attribute in `base.html`
- [x] `./dev.py i18n` command (runs `makemessages -l de -l en` + `compilemessages`)
- [x] `inquiry_confirm_delete.html` — reference implementation showing the pattern

## Contributor workflow (scavenger hunt)

Once the scaffold is merged, each template becomes a self-contained issue:

1. Add `{% load i18n %}` to the template (alongside `{% load static %}`)
2. Wrap UI strings: `"Löschen"` → `{% trans "Löschen" %}`
3. For strings with variables: use `{% blocktrans %}…{% endblocktrans %}`
4. Run `./dev.py i18n` — this extracts strings into `locale/de/django.po`
   and `locale/en/django.po`, then compiles them
5. Add the English `msgstr` values in `locale/en/LC_MESSAGES/django.po`
6. Run `./dev.py i18n compilemessages` again, test in browser

See `inquiry_confirm_delete.html` as the reference implementation.

## Not in scope

- Email body templates (already have DE/EN variants)
- PDF templates (language is per-invoice, already handled)
- URL slugs (already English after P-038)

## Notes

- German is the primary language — DE strings stay as-is in templates; EN needs translation in `.po` files
- `.po` source files are committed; `.mo` compiled files are generated locally (gitignored)
- `--no-wrap` flag on `makemessages` keeps long strings on one line (easier diffs)
- Language switching uses `Accept-Language` header via `LocaleMiddleware` — no URL prefix needed
- `set_language` view at `/i18n/set_language/` available for an explicit language switcher if wanted later
